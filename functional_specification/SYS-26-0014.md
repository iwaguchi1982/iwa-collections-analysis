# Run比較ワークスペースの実装 (SYS-26-0014)

## 1. 背景
多数の解析結果（カットオフ違い、共変量違い、コホート違い等）を横断的に比較し、解析条件の変更が結果に与えた影響を構造的に把握するためのワークスペースが必要である。
本機能により、研究の試行錯誤プロセスを記録し、論理的な比較説明を容易にする。

## 2. 目的
- 複数 Run の条件および結果の差分をサイドバイサイドで確認する。
- どの変更が結果の有意性や効果量に寄与したかを明確にする。
- 解析の再現性と履歴管理の効率を向上させる。

## 3. 基本設計方針
- 各解析画面から「一瞬」で現在の状態を保存（Snapshot）可能にする。
- 軽量なメタデータ中心の保持により、メモリ負荷を最小限に抑える。
- 数値差分（Δ）の自動計算と、定性的な変化要約（Effect preserved/attenuated等）を提供する。

## 4. 機能詳細
### 4.1 Run Snapshot & Management
- ユーザー任意のラベル（例: "50th / Full Adjustment"）を付けて現在の解析結果を保存。
- 最新の 2件以上の Run を比較可能。

### 4.2 Metadata & Result Comparison
- コホート、遺伝子、カットオフ、共変量セットの不一致を自動検出。
- HR、p-value、カットオフ値の差分（Δ）をテーブル表示。

### 4.3 Issue & Summary Comparison
- **Analytical Issues**: New（新規発生）、Resolved（解消）、Persistent（継続中）に分類して比較。
- **Automated Summary**: 2つの Run の関係性を「Effect direction is preserved but HR amplified」のように具体的に要約。

## 5. UI構成
- **Run Selector**: 比較対象とする 2つの Snapshot を選択。
- **Comparison Table**: 条件、コア結果、警告の不一致をハイライト表示。
- **Comparison Summary**: 数値的変化を伴う自然言語による比較要約。

## 6. 実装上の注意
- 生データや巨大な DataFrame を保存対象にせず、要約統計量のみを保存する。
- 数値の差分だけでなく、カットオフのパーセンタイル移動量などの文脈も考慮した解釈を生成する。
