# Analysis Missions と動的 UI ルーティング (DEV-26-0003)

## 1. 背景
本システムは多機能化に伴い、タブ（Tab）の数が増加している。ユーザーの目的（バイオマーカー探索、標的評価、薬物再目的化など）によって、頻繁に使用するタブや推奨される作業順序が異なるため、UI を動的に最適化する必要がある。

## 2. 目的
- ユーザーの目的（Mission）に応じた「推奨パス（Route）」を提示し、関連するタブを優先表示・並べ替えする。
- 初心者ユーザーがどの機能から順に使えばよいかのガイドを提供し、迷いをなくす。

## 3. 基本設計方針
- `ROUTES` 定数により、各ミッションに対応するタブ名とその順序、内部 ID を定義する。
- Streamlit の `st.tabs` を動的に生成し、選択されたルートに基づいてコンテンツをルーティングする。

## 4. 機能詳細
### 4.1 Mission Selection
- アプリ上部で 4つの主要ルート（Default, Route A, B, C）を選択可能。
- `st.session_state` により、選択されたルートをアプリ横断で維持。

### 4.2 Dynamic Tab Rendering
- `tab_keys` リストに基づき、対応するビュー関数（`render_deg_view`, `render_auto_discovery` 等）を正しいタブオブジェクト内に描画。

### 4.3 Contextual Hinting
- ルートごとに、その目的（例: 「長期生存 vs 短期生存の比較から始めてください」）を説明する Info ボックスをトップに表示。

## 5. UI構成
- **Mission Selector (Radio buttons)**: アイコン付きの分かりやすいミッション選択 UI。
- **Reordered Tabs**: ルートに応じて「DEG」が最初に来たり、「Auto Discovery」が最後に来たりと、最適化されたタブ配置。

## 6. 実装上の注意
- タブの切り替えが発生した際、入力フォームの状態（Session State）がリセットされないよう、各ウィジェットに一貫した `key` を指定し続ける。
- 全てのタブを網羅する「Default」ルートを常に提供し、上級者が自由に機能を行き来できる余地を確保する。
