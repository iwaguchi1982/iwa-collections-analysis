# Dataset Registry とメタデータ管理 (DEV-26-0002)

## 1. 背景
本システムでは、個別の TCGA コホート、複数のコホートを束ねた Pan-Cancer データ、さらにはユーザーがインポートしたカスタムデータが混在する。これらを一貫したインターフェースで読み込み、管理するための抽象化層が必要である。

## 2. 目的
- データセットの種類（TCGA, PANCAN, CUSTOM）によらず、統一されたメタデータ形式でアプリ側にデータを提供する。
- データ読み込み時のフォーマット変換（Wide/Long）や、症例 ID の正規化（TCGA バーコードの 12桁切り出し等）を隠蔽する。

## 3. 基本設計方針
- `DatasetSpec` データクラスにより、各データセットの名前、種類、パス、含まれるメンバーを定義する。
- `list_dataset_specs` 関数でディレクトリ構成から動的に利用可能なデータセットを列挙する。
- 重いデータ読み込みには Streamlit の `cache_data` を適用し、メモリ効率と応答速度を両立する。

## 4. 機能詳細
### 4.1 TCGA Standard Loader
- cBioPortal 形式の clinical/expression ファイルを読み込み、生存時間情報の列名を標準化（OS_MONTHS 等）。

### 4.2 Pan-Cancer Aggregator
- 複数のコホートを指定された接尾辞（例: `_pan_can_atlas_2018`）を元に動的にグループ化。
- 結合時に各症例 ID にがん種プレフィックス（例: `BRCA:`）を付与し、ID の重複を防止。

### 4.3 Patient ID Normalization
- 文字列の長さ、ケース（大文字・小文字）、ハイフンの有無などのゆれを吸収。

## 5. UI構成
- **Dataset Selector (Sidebar)**: ユーザーが直感的にデータセット（および Pan-Cancer モード）を選択できるドロップダウン。
- **Dataset Summary Card**: 読み込み完了後に表示される、症例数、イベント数、がん種構成の統計。

## 6. 実装上の注意
- メモリ消費を抑えるため、未使用のデータフレーム（CNA, Mutation 等）は必要になるまで読み込まない「遅延ロード（Lazy Loading）」を意識する。
- インポートされたカスタムデータにおいて `PATIENT_ID` が見つからない場合の「推測ロジック（第一列の使用等）」のスループットを維持する。
