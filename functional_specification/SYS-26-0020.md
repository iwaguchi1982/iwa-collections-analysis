# 解析失敗理由の説明強化機能の実装 (SYS-26-0020)

## 1. 背景
生存時間解析（Cox モデル、KM 法、PSM 等）は、データの構造（イベント数、群の偏り、欠測、共線性等）によって実行不能になったり、結果が極端に不安定になったりすることが多い。現状の「エラー表示」だけでは、研究者が次に何をすべきか判断しにくい。

## 2. 目的
- 解析が失敗、あるいは信頼性が低いと判断された原因を、統計専門用語ではなく「研究者向けの具体的な説明」として提供する。
- 解析を成功させるための具体的な対処（アクション）を提案し、試行錯誤のプロセスを支援する。

## 3. 基本設計方針
- 内部のガードレール（Guardrails）が検知したフラグを、階層的なメッセージ体系に変換して表示する。
- 問題の重要度を「Critical（解析不可）」「Warning（信頼性に疑義）」「Notice（解釈に注意）」の 3段階で提示する。
- 単なるエラー提示にとどまらず、「カットオフを緩める」「共変量を減らす」等の推奨オプションを提示する。

## 4. 機能詳細
### 4.1 Condition Diagnosis Engine
- **Event Scarcity**: 全体または特定群でのイベント数不足の検知。
- **Linear Separation**: Cox モデルにおける特定群のイベントなし（準完備分離）の検出。
- **Multicollinearity**: 投入した共変量間の強い相関によるモデル不安定化。
- **PH Violation**: 比例ハザード仮定の著しい逸脱。

### 4.2 Prescriptive Suggestions
- 「症例数が足りないため、サブグループ解析ではなく全体解析に戻ることを推奨」といった文脈に応じたアドバイスの生成。
- 調整が多すぎる場合の「共変量の優先順位（推奨セット）」の提示。

### 4.3 Documentation Integration
- これらの警告や制約（Limitations）を、自動生成される解析レポート（SYS-26-0010）に機械的に記録する。

## 5. UI構成
- **Smart Warning Panel**: 解析画面上部に表示される、折りたたみ可能なアラート詳細。
- **Troubleshooting Guide**: エラー発生時に起動する、対話形式、またはヒント形式の解決支援。
- **Model Health Badge**: 現在の解析設定の「健康状態（安定性）」をスコアや色で表示。

## 6. 実装上の注意
- ユーザーに「解析が間違っている」と感じさせるのではなく、「データの制約（Nature of data）」であることを強調し、建設的な改善を促す表現を選択する。
- ライブラリ（lifelines, statsmodels 等）が吐き出す生のエラーメッセージ（Singular Matrix 等）を直接見せず、必ず翻訳（ラップ）して表示する。
- 解析を強制実行できる「Override」モードを用意する場合でも、そのリスクを十分に警告する。
