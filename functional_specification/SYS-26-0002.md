# ブートストラップによる頑健性評価 (SYS-26-0002)

## 1. 背景
解析結果（HR、p値、最適カットオフ等）が、対象となる患者集団のわずかな変動に対してどの程度安定しているかを評価することは、研究の再現性を担保する上で極めて重要である。

## 2. 目的
- 患者再標本化（Bootstrap Resampling）を通じて、主要結果の安定性を定量化する。
- 特定の症例に過度に依存した「外れ値的所見」を早期に検出し、誤解を招く解釈を防止する。

## 3. 基本設計方針
- 患者標本を重複を許して再抽出（n=100〜500）し、同一解析を並列実行する。
- 得られた HR や p 値の分布を可視化し、安定性を「Stable / Unstable」などのラベルで提示する。
- 1-Gene および 2-Gene 解析の両方に適用可能な共通ロジックを実装する。

## 4. 機能詳細
### 4.1 Bootstrap Engine
- `src/bootstrap.py` による並列サンプリング処理。
- 再抽出回数のユーザー指定（デフォルト 100回）。

### 4.2 Stability Distribution Analysis
- 再抽出された標本群における HR の 95% 信頼区間の再評価。
- log-rank p 値が有意水準（0.05等）を下回る頻度の算出。
- 最適カットオフ値（Cutoff Search 使用時）の分布と最頻値の特定。

### 4.3 Robustness Classification
- 分布の広がり（CV値等）に基づく安定性判定基準の適用。
- 有意性が維持される割合（Success Rate）の提供。

## 5. UI構成
- **Bootstrap HR Histogram**: 再抽出による HR の分布と、オリジナルの点推定値の対比。
- **Stability Metrics Badge**: 有意維持率や分布の収束状況をバッジで表示。
- **Warning Messages**: 分布が二値化（bimodal）する場合や、CI が極端に広い場合の警告。

## 6. 実装上の注意
- 計算負荷が高いため、デフォルトではオフ、またはバックグラウンド処理での実行を検討。
- 標本数が極端に少ないコホートではブートストラップ自体が不安定になるため、最小 N によるガードレールを設ける。
