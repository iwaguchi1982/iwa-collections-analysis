# 外部知識源 (OpenTargets) の Live/Mock 統合 (DEV-26-0004)

## 1. 背景
バイオマーカー候補を評価する際、その遺伝子の創薬可能性（Druggability）や既知の薬剤情報を外部知識源（OpenTargets 等）から取得する必要がある。一方、ネットワーク制限のある環境や、開発効率の観点から、ローカルの模擬データ（Mock）を使用する選択肢も重要である。

## 2. 目的
- 外部 API（OpenTargets GraphQL API）からのリアルタイム取得と、ローカルの SQLite/JSON 模擬データからの取得を透過的に切り替える。
- 外部接続が不安定な場合でも、解析フローが中断されないようなフォールバック機構を提供する。

## 3. 基本設計方針
- 一括取得用の関数（`get_target_profile`, `get_known_drugs`）に `use_mock` フラグを設け、ソースを動的に切り替える。
- API レスポンスのスキーマを標準化し、Live と Mock で同一のデータ形式を返却するように実装する。

## 4. 機能詳細
### 4.1 Live API Client
- OpenTargets GraphQL 端点へのクエリ実行。
- 通信エラーやタイムアウト時の例外処理とキャッシュ管理。

### 4.2 Mock Database
- 内部的なテストデータセット（主要ながん関連遺伝子数百個程度）の保持。
- `is_mock_enabled` フラグによる、オフラインモードでの動作保証。

### 4.3 Target Profile Consolidation
- 分子機能（Kinase, Receptor 等）、細胞内局在（Plasma membrane 等）、創薬フェーズの情報を一つのデータフレームに統合。

## 5. UI構成
- **Data Source Selector (Advanced Settings)**: 「Mock Database (Internal)」と「Live API (External)」を選択するラジオボタン。
- **Source Indicator**: Druggability セクションのヘッダー付近に、現在どのソースからデータを取得したか（API vs Local）を明記。

## 6. 実装上の注意
- API リクエストにはレート制限（Rate limit）があるため、短時間に大量の遺伝子を照会する場合はバッチクエリを使用するか、結果を適切にキャッシュする。
- セキュリティ上の理由で外部 API へのアクセスが遮断される環境では、自動的に Mock モードにフォールバックするロジックを検討する。
