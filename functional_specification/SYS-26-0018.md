# サブグループ一括解析機能の実装 (SYS-26-0018)

## 1. 背景
本アプリでは、多面的な解析が可能になっているが、研究や論文化においては特定集団（病期、性別、年齢等）での効果の一貫性（Robustness）や効果修飾（Interaction）の評価が不可欠である。
本機能は、複数サブグループを一括走査し、Forest Plot や Interaction p-value を用いて解釈を効率化するダッシュボードを提供する。

## 2. 目的
- サブグループごとの効果量を一覧化し、一貫性を評価する。
- 交互作用（Interaction）の統計的有意性を確認する。
- 症例不足のサブグループを適切に除外・警告する。
- 論文化等に活用可能な Forest Plot を生成する。

## 3. 基本設計方針
### 3.1 総合方針
1つの候補に対して複数サブグループを一括で実行し、HR・95% CI・交互作用をまとめて確認できる専用ビューを実装する。

### 3.2 設計原則
- 保存された 1-Gene 解析の条件（遺伝子、カットオフ、共変量）をシームレスに引き継ぐ。
- Forest Plot を中心とした視覚的理解を優先する。
- 症例数（N）およびイベント数を必ず併記し、信頼性を担保する。

## 4. 機能詳細
### 4.1 Eligibility Check (適格性判定)
各サブグループが解析に十分なデータ（Nおよびイベント数）を有しているかを自動判定します。単なる判定だけでなく、以下の具体的な「除外理由」を提示します：
- **too few patients**: サンプルサイズ不足（N < 20）
- **too few events**: イベント数不足（Events < 5）
- **one arm too imbalanced**: ターゲット群の偏りが極端（片群 5% 未満）
- **no variation in target group**: 全員が同一群（WTのみ等）
- **missing subgroup label**: サブグループラベルが欠測

### 4.2 Interaction Test (交互作用検定)
ターゲット変数とサブグループ因子の交互作用項を含む Cox モデルを実行し、効果修飾の統計的有意性を評価します。
モデルの安定性を担保するため、以下の「ガードレール」を適用します：
- 選択可能な解析軸を **Sex, Stage, Age group, Subtype** 等の推奨候補に優先制限。
- サブグループのレベル数が多すぎる（>6）場合は、不安定化を防ぐためスキップ。
- 各層に最低 1 以上のイベントが必要。
- モデル収束不全や特異行列を検出し、warning を出力。

### 4.3 Consistency Classification (一貫性分類)
解析結果を標準化されたフィールド（HR, CI, p-value, status, note）で出力し、以下のカテゴリに自動分類します：
- **Broadly Consistent**: 主要な層で方向が一致し、交互作用検定に成功し有意でない。
- **Heterogeneous (Potential Subgroup-Specific Effect)**: 交互作用が統計的に有意。
- **Directionally Inconsistent**: 層によって効果の方向（HR > 1 vs < 1）が逆転。
- **Insufficient Data**: 解析可能な層が存在しない。

## 5. UI構成
- **Subgroup Selector**: 推奨される臨床パラメータを優先表示。
- **Interpretation**: 交互作用 p-value の妥当性と警告、自動生成された解釈テキスト。
- **Subgroup Forest Plot**: HR と 95% CI の可視化。
- **Detailed Results Table**: 標準化された項目（N, Events, HR, P-value, Status, exclusion reason）を一列に表示。

## 6. 実装上の注意
- 症例不足の層を無理にプロットしない。
- 交互作用検定の結果なしに「サブグループ固有の効果」と断定するのを避ける。
- 解析条件（Omics種別等）が Analysis タブと一致していることを保証する。
