# 複数コホート一括検証 (Meta-Analysis) Dashboard (DEV-26-0005)

## 1. 背景
特定のがん種で見つかったバイオマーカーが、他のがん種や別のデータセットでも普遍的な意義を持つかを評価する（汎がん解析・一括検証）ことは、臨床的価値を証明するために不可欠である。データセットごとに手動で解析を繰り返す手間を省く必要がある。

## 2. 目的
- 同一の解析条件（ターゲット遺伝子、カットオフ値）を選択した複数のコホートに一括適用し、比較を容易にする。
- 各コホートのハザード比（HR）を Forest Plot として可視化し、一貫した予後予測能があるかを俯瞰可能にする。

## 3. 基本設計方針
- 全データセットを並列または逐次処理し、統一されたフォーマットで統計量を収集する。
- 各データセット内の発現レベルの違いを吸収するため、絶対値（Linear）ではなく「コホート内パーセンタイル」または「Z-score」による条件の転移（Transfer）をサポートする。

## 4. 機能詳細
### 4.1 Multi-Cohort Process Loop
- 選択された各データセット（`df_clin`, `df_exp`）を `load_dataset_cached` で順次呼び出し、Cox 解析を実行。

### 4.2 Threshold Normalization
- **Percentile Mode**: 各コホートの上位 X% を High 群とする。
- **Z-score Mode**: 各コホート内での Z-score > 1.0 等を High 群とする。

### 4.3 Meta-Synthesis Logic
- 解析成功したコホートの数、有意（p < 0.05）となった割合、効果の方向性（HR > 1 vs < 1）の比率を自動集計。

## 5. UI構成
- **Meta-Analysis Workspace**: 対象コホート、遺伝子、しきい値の指定エリア。
- **Meta-Forest Plot**: 複数の HR と 95% CI を並べたグラフ。
- **Synthesis Summary**: 「N個中 M個で有意な相関を確認」といったサマリー文章の提示。

## 6. 実装上の注意
- 複数のデータセットを同時に扱うため、計算処理中のプログレスバー表示を徹底し、ユーザーの不安を解消する。
- 症例数が極端に少ない、あるいは遺伝子が存在しないコホートは「スキップ」として扱い、エラーの詳細を別枠で報告する。
