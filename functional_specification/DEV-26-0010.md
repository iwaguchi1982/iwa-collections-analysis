# 解析信頼性のための Reliability Guardrails (DEV-26-0010)

## 1. 背景
生存時間解析における「p < 0.05」という結果は、症例数（N）やイベント数（死亡数）が極めて少ない場合、統計的安定性を欠き、偶然に左右されやすくなる。ユーザーが数学的な不備に気づかず結論を下すのを防ぐためのガードレールが必要である。

## 2. 目的
- 解析開始前、または計算直後に、データの状態が統計解析の前提条件（ルール）を満たしているかを判定する。
- 信頼性のレベルを「信号機（青・黄・赤）」形式で表示し、重大な不備がある場合には解析を停止、あるいは強い警告を発する。

## 3. 基本設計方針
- イベント数（Events）および群ごとの症例数に対するしきい値（Heuristic thresholds）を定義する。
- 推定されるパラメータ数（共変量数）に対するイベント数の比率（Events-per-parameter; EPP）を評価する。

## 4. 機能詳細
### 4.1 Status Classification
- **OK (Green)**: 症例・イベントが十分にあり、EPP > 10 を満たす。
- **CAUTION (Yellow)**: イベント数が境界（15〜30）にある、あるいは一部の群が少数。
- **DANGER (Red)**: イベント数が極少（<15）、または EPP < 5。結果の解釈を推奨しない。

### 4.2 Heuristic Rules
- 「いずれかの群でイベント数が 0 の場合」は完全分離（Complete separation）となり最尤法が収束しないため DANGER と判定。
- 4群比較（2-Gene）においては、分割による症例激減を検知。

### 4.3 Contextual Warning Messages
- 単なる「エラー」ではなく、「このイベント数では査読で信頼性を指摘される可能性が高いです」といった研究者向けの助言を付加。

## 5. UI構成
- **Reliability Status Box**: 解析結果のすぐ上（またはサイドバー）に表示される色付きのステータスバナー。
- **Summary Metrics Overview**: N, Events, EPP, Min group N 等の重要指標のクイックビュー。

## 6. 実装上の注意
- 厳格にしすぎると何も解析できなくなるため、DANGER（赤）は「統計的にほぼ破綻している状態」に限定し、CAUTION（黄）で「慎重な解釈」を促す運用とする。
- ガードレール自体のしきい値を必要に応じて調整（または無効化）できる「Expert mode」の導入を検討する。
