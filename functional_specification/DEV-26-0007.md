# 予後結果ベースの探索解析 (DEG & PCA) Dashboard (DEV-26-0007)

## 1. 背景
「特定の遺伝子が予後に効くか」を確認する順方向の解析に対し、まずは「生存期間の長い群と短い群を分け、その間で最も発現が異なる遺伝子を網羅的に探す」という逆方向の探索（Discovery）が必要な場合がある。

## 2. 目的
- 実際の生存期間データ（例: 5年以上生存 vs 2年以内に死亡）に基づいて、患者を 2つの Outcome グループに分類する。
- その 2群間における変動遺伝子（DEG）や、遺伝子発現パターンによる集団の分離度（PCA）を可視化する。

## 3. 基本設計方針
- ユーザーが「Good Outcome」と「Poor Outcome」の定義（月数、打ち切り含むか否か）を自由に変更できるようにする。
- 大規模な発現データに対して、高速な T-検定を実行し、Volcano Plot を生成する。

## 4. 機能詳細
### 4.1 Discovery Grouping Logic
- `OS_MONTHS` と `is_dead` を用いた動的な患者ラベリング。
- マージナルな（どちらの群にも属さない、中間の予後の）症例の自動除外。

### 4.2 DEG Engine
- Log Fold Change（FC）と p-value の一括計算。
- 指定された遺伝子リスト、またはランダムサンプリングされた遺伝子集合に対する高速演算。

### 4.3 PCA (Principal Component Analysis)
- 特徴量（遺伝子発現）を用いた 2次元の主成分分析。
- 生成された Outcome グループによる色分け表示と、集団の分離性の視覚的評価。

## 5. UI構成
- **Outcome Threshold Selector**: 「長期生存」と「短期死亡」のしきい値を指定する数値入力エリア。
- **Volcano Plot Viewer**: 有意な変動遺伝子（Up/Down）をハイライトした散布図。
- **Interactive Top Genes Table**: p値順、FC順にソート可能な変動遺伝子リスト。

## 6. 実装上の注意
- 全遺伝子（数万個）を対象とする場合、計算時間がかかるため、プロトタイプ版では「ランダムサンプリング（Discovery モード）」または「注目パスウェイ」に限定する。
- データのスケーリング（Log2 変換の有無）を検知し、適切な Fold Change 計算を行う。
- PCA 計算時の特異値分解（SVD）でのエラーハンドリング（不定値の除外等）を行う。
