# 傾向スコアマッチング (PSM) の中核実装 (DEV-26-0006)

## 1. 背景
観察データを用いた予後解析において、注目遺伝子の発現群（High/Low）と臨床背景因子（年齢、性別、病期等）が交絡している場合、得られた HR が遺伝子そのものの効果なのか、背景因子の差なのかを区別できない。これを解決するために PSM による背景調整が必要である。

## 2. 目的
- ロジスティック回帰を用いて「特定の遺伝子発現群に属する確率（傾向スコア）」を推定する。
- 傾向スコアが近似する症例を 1:1 または 1:N でマッチングさせ、仮想的なランダム化比較試験に近い集団を生成する。

## 3. 基本設計方針
- 各症例の臨床共変量を説明変数、発現群（High=1 / Low=0）を目的変数としたモデルを構築する。
- マッチングアルゴリズムとして最近傍法（Nearest Neighbor Matching）を採用し、カリパー（許容幅）の設定を可能にする。

## 4. 機能詳細
### 4.1 Propensity Scoring Model
- `statsmodels` または `scikit-learn` によるロジスティック回帰の実行。
- 各症例への propensity score の付与。

### 4.2 Matching Algorithm
- 重複を許さない 1:1 マッチング処理。
- マッチしたペアの生存時間データの抽出と新規データフレームの生成。

### 4.3 Balance Assessment
- マッチング前後での各共変量の平均値差や分散の改善度（SMD）の算出。

## 5. UI構成
- **PSM Configurator**: 調整に加えたい（モデルに投入する）臨床因子の選択パネル。
- **Balance Matrix Table**: 各変数の SMD が 0.1 以下（一般的にバランス良好とされる基準）に収まったかを確認する表。
- **Matched KM Curve**: マッチング後の集団のみを用いた KM 曲線と log-rank 検定結果。

## 6. 実装上の注意
- マッチングによって症例数が減少するため、元の解析（Unadjusted）との N 数の差を常に強調して表示する。
- マッチしたペアの依存性を考慮し、検定には層別化された Cox モデルや、適切な分散推定を用いる。
