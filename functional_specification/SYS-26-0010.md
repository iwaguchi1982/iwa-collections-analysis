# 解析レシピと出力の自動レポート化 (SYS-26-0010)

## 1. 背景
本システムは dataset、gene、cutoff、covariates などのパラメーターが多岐にわたるため、どの条件でどの結果が出たかを手作業で記録するのは極めて非効率であり、ミスが入りやすい。

## 2. 目的
- 各 Run の解析条件、統計指標、生成された図をパッケージ化し、自動保存する。
- 研究の再現性を保証し、論文化の際のスプリメント資料作成時間を大幅に短縮する。

## 3. 基本設計方針
- 解析結果（HR, p値等）とメタデータを JSON 形式で永続化する。
- 保存したレシピからの再実行機能を備え、履歴管理を実現する。
- 多様なフォーマット（CSV, PDF, HTML）でのレポート出力を可能にする。

## 4. 機能詳細
### 4.1 Metadata Recording
- 解析日時、データセット名、フィルタ条件、遺伝子 ID、カットオフ値。
- 使用された統計モデル（Cox, RMST 等）と共変量リスト。

### 4.2 Result Artifact Bundling
- KM Plot, Forest Plot などの画像ファイルとの紐付け。
- 統計指標（N, Events, HR, CI, P, Q）のサマリーテーブル。

### 4.3 Export Engine
- 学術誌投稿用の Supplemental Table 形式の CSV 生成。
- ミーティング共有用の要約 PDF レポートの作成。

## 5. UI構成
- **Analysis History / Recipe View**: 過去に実行した解析のカード型一覧。
- **Report Preview**: 出力されるレポートの内容を確認・編集する画面。
- **Export Settings**: 出力フォーマットや含める図の選択オプション。

## 6. 実装上の注意
- 保存対象に生データ（巨大な DataFrame）を直接含めず、再現に必要なキー情報と要約統計量に限定してストレージを節約する。
- 解析条件に変更があった場合（データの更新等）、履歴と矛盾が生じないようバージョン管理を行う。
