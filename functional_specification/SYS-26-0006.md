# 外部コホート取り込みの一般化 (SYS-26-0006)

## 1. 背景
Discovery 解析で見出した候補を検証（Validation）するには、GEO や自前解析データなどの多様な外部データセットを迅速に取り込み、標準フォーマットに変換する工程が必要である。

## 2. 目的
- 学術データベース（GEO, ArrayExpress 等）や自前 RNA-seq/Clinical データのインポート導線を簡便化する。
- データの形式（Wide/Long）や ID 体系の違いを吸収し、即座に予後解析が可能な状態にする。

## 3. 基本設計方針
- ユーザーによる「列マッピング UI」を提供し、臨床情報（OS, DFS, Event）を特定しやすくする。
- インポート時にデータ品質の自動診断（QC）を実行し、解析の不備を未然に防ぐ。
- Gene ID（Symbol, Entrez, Ensembl）の自動変換と不一致ログの保持。

## 4. 機能詳細
### 4.1 Flexible Uploader
- CSV / TSV 形式のカスタムデータアップロード。
- 発現マトリックスと臨床情報の結合処理。

### 4.2 Clinical Mapping UI
- ユーザーが「どの列が生存時間か」「どの列が生存ステータスか」を指定する直感的な UI。
- 時間単位（月、日、年）の自動正規化。

### 4.3 Automated QC & Reporting
- 欠測値、外れ値、症例重複の自動検出。
- 遺伝子発現値の分布（Normalization 状況）のサマリ提示。

## 5. UI構成
- **Import Wizard**: ファイル選択からマッピング、QC までのステップ形式 UI。
- **Mapping Dashboard**: 既存の臨床項目とアップロード項目の紐付けプレビュー。
- **Data health check**: 「OK / Warning / Critical」の判定とともに問題箇所をリストアップ。

## 6. 実装上の注意
- 遺伝子名の別名（alias）対応により、マッチング率を最大化する。
- 巨大なドロップダウンからの列選択を避け、検索・サジェスト機能を持たせる。
- 個人情報（PII）が含まれやすい外部データの取り扱いに配慮（または注意喚起）を行う。
