# 図表品質管理機能の実装 (SYS-26-0015)

## 1. 背景
学術論文の投稿や学会発表、資料作成において、解析結果のグラフ（KM Plot, Forest Plot 等）が「そのまま使える品質」であることは極めて重要である。フォントの統一性、余白、配色の配慮など、細部への手間を削減する必要がある。

## 2. 目的
- 学術誌の要求水準を満たす、一貫性のある高品質な図表出力を自動化する。
- 出力時のカスタマイズ（サイズ、解像度、配色のバリアフリー化）を容易にし、図の整形作業を最小化する。

## 3. 基本設計方針
- 全ての図で共通のテーマ（Theme/Style）を適用し、フォントサイズや線の太さを統一する。
- 論文投稿用（TIF/PDF）、スライド用（High-res PNG）、Web 用それぞれのプリセットを用意する。
- 凡例や注釈（P-value, HR, N 数）の配置、リスクテーブルの体裁を最適化する。

## 4. 機能詳細
### 4.1 Unified Design System
- アプリ全体の図のデザインガイドライン（カラーパレット、フォント指定）の実装。
- 色覚バリアフリー（Color blind friendly）な配色の標準化。

### 4.2 High-Resolution Export Engine
- 300-600 DPI を保証する高画質出力。
- SVG, EPS などのベクター形式への対応（将来拡張含む）。

### 4.3 Dynamic Annotation
- 図内の注釈テキスト（例: "Cutoff=Top 25%"）の自動生成と位置調整。
- フォントサイズの一括変更スライダー。

## 5. UI構成
- **Plot Customizer Sidebar**: 線種、シンボル、フォント、カラーパレットのリアルタイム編集。
- **Export Previews**: ジャーナル別などの出力サンプルプレビュー。
- **Annotation Manager**: 図内のラベルや脚注（Footnote）の編集 UI。

## 6. 実装上の注意
- matplotlib や plotly のデフォルト設定を隠蔽し、本システム独自の `StyleManager` を通じて全描写を制御する。
- 日本語フォントの文字化け（豆腐現象）や、PDF へのフォント埋め込み問題に対するケアを確実に行う。
- 図自体の品質だけでなく、軸ラベルが物理単位（months, years）を含んでいるか等のメタデータの正確性を保証する。
