# Gene Signature (Multi-Gene) 解析ロジック (DEV-26-0001)

## 1. 背景
単一遺伝子の発現量のみならず、特定の生物学的プロセスやパスウェイに関与する複数の遺伝子群（Gene Signature）を統合して解析したいというニーズがある。個々の遺伝子の変動を合算・標準化することで、より安定したバイオマーカーとしての評価が可能になる。

## 2. 目的
- ユーザーが指定した複数の遺伝子リストから、症例ごとの「Signature Score」を算出する。
- スコアを擬似的な 1つの変数をとして扱い、既存の 1-Gene 解析パイプライン（KM, Cox 等）をそのまま適用可能にする。

## 3. 基本設計方針
- 各遺伝子の発現値を Z-score（平均 0, 標準偏差 1）に変換した上で合算し、さらにその合算値を Z-score 化する。
- 指定された遺伝子のうち、データセット内に存在しない、あるいは発現変動がない遺伝子を自動的に除外し、有効な遺伝子のみで算出する。

## 4. 機能詳細
### 4.1 Scoring Algorithm
- 1. 各遺伝子 $g_i$ について、全症例での Z-score $z_i$ を計算。
- 2. 症例 $j$ における未加工スコア $S'_j = \sum z_{ij}$。
- 3. 全症例にわたる $S'$ を再度 Z-score 化して最終的な $S_j$ とする。

### 4.2 Handling Missing Genes
- 指定リスト内の遺伝子が発現マトリックスに存在するかをチェック。
- 存在比率（Found / Missing）をメタデータとして保持し、UI に表示する。

### 4.3 Integration
- 算出されたスコアを新しい列としてデータフレームに追加し、`render_1gene_analysis` 等の既存 UI コンポーネントに注入。

## 5. UI構成
- **Signature Input Manager**: カンマ区切り、または改行区切りでの遺伝子リスト入力エリア。
- **Found/Missing Status List**: どの遺伝子が解析に含まれたかのステータス表示。
- **Distribution Plot**: 生成された Signature Score の分布確認（ヒストグラム）。

## 6. 実装上の注意
- 複数のデータセット（Meta-Analysis）に適用する場合、各データセットの平均・標準偏差で各個に Z-score 化を行い、バイアスを抑制する。
- 逆方向に働く遺伝子（抑制因子）が含まれる場合の「符号反転」機能については将来拡張とする。
