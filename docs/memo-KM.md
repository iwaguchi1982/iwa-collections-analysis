# Kaplan-Meier

## KM(Kaplan-Meier)とは何か。
KM曲線は、ひとことで言うと、**「時間がたつにつれて、イベントが起きずに残っている割合」を描くグラフ**

ここでいうイベントは、解析目的によって変わります。
- 全生存期間（OS）なら「死亡」
- 無増悪生存期間（PFS）なら「増悪または死亡」
- 再発までの期間なら「再発」
- 装置故障解析なら「故障」
KM は、本質的には**“生存”専用のグラフではなく、時間経過とイベント発生を扱う方法**です。
医学で多用されるので survival analysis と呼ばれますが、考え方自体はもっと一般なもの。

## なぜ普通の平均や割合ではだめなのか

臨床データでは、全員を最後まで追跡できるとは限りません。

たとえば患者さんごとに：
- 5年追えた人
- 2年で転院して以後不明な人
- まだ生きていて観察継続中の人
が混ざります。

このとき単純に
- 死んだ / 死んでいない
- 平均生存月数
だけを見ると、追跡期間の違いが無視されてしまいます。

KMはそこを補うために、
**「その人が観察できたところまでは情報として使う」**
というやり方をします。
この考え方が survival analysis の中心です。

## KM曲線は何を表しているのか
縦軸と横軸をまず明確にします。

横軸：時間
例：診断からの月数、治療開始からの月数、登録からの月数  
縦軸：その時点までイベントが起きていない確率  
これを survival probability と呼びます

たとえば縦軸が 0.8 なら、その時点までイベントが起きていない人の割合が約80%
という意味です。

## 曲線が階段状になる理由
KM曲線は、なめらかな線ではなく階段状になります。

理由は簡単で、
**イベントが起きた時点でだけ下がる**からです。  
- 誰もイベントを起こしていない期間 → 横ばい  
- 誰かがイベントを起こした時点 → ストンと下がる  

この「下がり」が多いほど、イベントが多く起きていることを意味します。

## 打ち切り（censoring）とは何か
KMを理解するうえで最重要概念です。

打ち切りとは、

**“その人について、そこまではイベントが起きていないことは分かるが、その先は分からない”**
という状態です。

たとえば：
- 研究終了時点でまだ生存
- 途中で追跡不能
- 他院へ転院
- 解析時点ではまだイベント未発生

この場合、その人は「イベントなし」とは断定できません。
ただし、「少なくともここまではイベントなし」は使えます。

KMはこの情報を組み込みます。

打ち切りのイメージ
```
ある患者が20か月追跡され、20か月時点でまだ生存していたがその後不明なら、
20か月までは“生存していた”情報を使う
20か月以降はその患者はリスク集合から外れる
```
という扱いです。

## KM曲線はどうやって作られるか
厳密な数式を全部覚える必要はありませんが、考え方は大事です。

各イベント時点で、  
**その時点まで生き残っていた人のうち、何人がイベントを起こしたか**  
を見ます。  

そして、  
**その都度の“生き残る割合”を順番に掛け算していく**  
ことで、曲線ができます。  

たとえば：
- 100人が観察中
  - ある時点で 10人イベント → その時点の生存は 90/100
  - 次の時点では打ち切りなどで 80人が対象、そのうち 8人イベント → 72/80
  - 累積すると 0.9 × 0.9 = 0.81
という感じです。

つまり KM は、単純な累積割合ではなく、各時点の条件付き生存確率を積み重ねたものです。

## 何を見るのか
KMを見るときは、ただ「線が低い・高い」だけでなく、次を見ます。

### 1. まずは群間の位置関係
たとえば
- High expression 群
- Low expression 群
を比べるとき、
- 上にある群 → イベントが起きにくい
- 下にある群 → イベントが起きやすい
と読みます。

OSなら、上にある群のほうが予後が良いと解釈されやすいです。
### 2. いつ差が開くか
差が
- 早期に開くのか
- 後半で開くのか
- 最初は同じで途中から分かれるのか
を見るのは大事です。

これは臨床的にかなり意味があります。

たとえば：
- 早期から差が開く → 初期リスク差が大きい可能性
- 後半で差が開く → 長期予後に効いている可能性
- 交差する → 単純なHRやlog-rankだけでは捉えにくい

### 3. 曲線の交差
KM曲線が交差すると、要注意です。
理由は、よく使う Cox モデルや log-rank 検定が、暗黙に
**「片方の群がだいたい一貫して高リスク/低リスク」**
という構図を想定していることが多いからです。

交差していると、
- 前半はAが悪い
- 後半はBが悪い
のようなことが起こり、単純な一言で言いにくくなります。

本アプリ上でも RMST を出しているのは、KM曲線が交差する場合の補助的な見方としてサポートする為です。

### 4. 後半の不安定さ
KMの右端に近づくほど、観察されている人数が減ります。
すると、後半の少数例で曲線が大きく動くことがあります。
つまり、**右端の見た目を信じすぎない**
ことが大事です。

たとえば最後のほうで急に1回落ちても、それが数人中1人なのか、数百人中1人なのかで意味は全く違います。

だから本当は number at risk（各時点で何人残っているか） を一緒に出すのがとても重要です。
