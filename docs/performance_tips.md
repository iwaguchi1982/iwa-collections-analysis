# 動作速度と再計算について

本アプリケーションは、対象遺伝子（Gene 1 / Gene 2）の変更や、しきい値（Percentile cutoff）の変更時に、データのマージ処理から生存時間解析（Cox回帰など）までの一連の重い計算を再実行します。

### なぜ画面全体が再計算されるのか？
- 生存時間解析は、グループ分け（HiHi, HiLo, LoHi, LoLo）の境界線が変わると、ハザード比やp値が大きく変動するため、毎回の再フィッティングが必須となります。
- そのため、サイドバーの設定スライダーを動かした直後は、計算完了まで数秒お待ちいただく場合があります。

### 動作を軽くする工夫 (st.fragmentの活用)
- 「Cut smoothness scan」など、全体データストラクチャに影響を与えない一部の機能のパラメータ調整においては、画面全体の再計算を防ぐための工夫（`@st.fragment`による部分描画）を導入しています。該当部分の操作時は非常に高速にプロットのみが更新されます。

### 将来の展望 (GPU利用による高速化設定)
- データの増大に備え、将来的には設定ファイル(`config.toml`)の `use_gpu` フラグを通じて、RAPIDS (cuDF/cuML) 等の強力なGPU支援ライブラリを有効化する仕組みを検討中です。
