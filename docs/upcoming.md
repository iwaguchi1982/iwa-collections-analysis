# 実装予定/検討中ログ
## 実装予定/検討中一覧
---
#### 背景
現状でも events per parameter や群ごとのイベント数に対する guardrail は整備されているが、
それだけでは「その結果がどの程度安定しているか」を十分に示せない。

#### 目的
HR、log-rank p、最適 cutoff、interaction 項などの主要結果が、患者再標本化に対してどの程度安定しているかを定量化する。

#### 実装内容案
- bootstrap resampling による HR 分布推定
- log-rank p 値の分布推定
- 最適 cutoff の再現頻度評価
- interaction 項有意性の安定性評価
- confidence interval の再推定
- “安定 / やや不安定 / 不安定” などの簡易ラベル表示

#### 研究上の利点
- 所見の頑健性を数値で示せる
- 過学習的な結果を見抜きやすい
- 査読や共同研究先への説明がしやすくなる

### SYS-26-0003）時間依存効果の本格対応
#### 背景
本システムには landmark analysis の導線があり、交差 KM などを意識した設計になっている。
ただし、遅効性や時間依存的な効果を持つケースでは、通常の Cox 比例ハザードモデルだけでは十分に説明できない場合がある。

#### 目的
時間依存効果を持つ候補についても適切に解析・可視化し、比例ハザード仮定違反時の代替的評価を可能にする。

#### 実装内容案
- time-varying coefficient model の導入
- landmark analysis の強化
- restricted mean survival time（RMST）の本格活用
- 3年 / 5年 survival probability の自動出力
- PH 違反時の代替解析推奨表示
- 通常 Cox と時間依存モデルの比較表示

#### 研究上の利点
- 交差 KM や遅効性効果を適切に扱える
- PH 仮定違反時の説明力が上がる
- 免疫系標的や複雑な予後パターンの解析に強くなる

### SYS-26-0007）機序探索への接続
#### 背景
本システムには DEG、PCA、druggability、known drugs など、機序探索に繋がる要素がすでにある。
ただし、予後差が見つかった後に「なぜそうなるのか」を生物学的に掘り下げる層はさらに強化できる。

#### 目的
予後関連候補の統計的発見から、生物学的背景や経路解釈まで一貫して辿れるようにする。

#### 実装内容案
- GSEA の実装
- Hallmark / KEGG / Reactome enrichment
- pathway activity score の算出
‐ 免疫細胞浸潤推定
- TME スコア表示
- DEG / PCA / subgroup 結果との連携
- 解釈補助コメントの表示

#### 研究上の利点
- “予後に効く” から “なぜ効くか” へ進める
- 考察や機序仮説の構築がしやすくなる
- 論文の生物学的深みを増せる

### SYS-26-0008）創薬・再目的化の現実味を上げる機能
#### 背景
druggability や known drugs は魅力的な要素だが、研究用途としては、薬剤情報をより構造化し、解析結果と定量的に結びつける余地がある。

#### 目的
予後関連候補や 2遺伝子候補を、創薬可能性・再目的化可能性の観点から優先順位付けできるようにする。

#### 実装内容案
- 疾患適応、作用機序、開発相、併用候補、耐性変異、バイオマーカー条件の構造化
- gene / gene pair と薬剤候補の紐付け
- triage スコアリング
- “既承認薬あり / 開発中 / 抗体薬適性あり” などの分類表示
- external knowledge source 連携強化
- 解析結果に基づく drug repurposing 候補提示

#### 研究上の利点
- 発見結果を治療仮説へ繋げやすい
- translational research 的な価値が上がる
- 共同研究や事業開発の観点でも使いやすい

### SYS-26-0009）PH仮定チェックと時間依存効果の正式実装
#### 背景
Cox 比例ハザードモデルは本システムの中心的解析法の一つだが、比例ハザード仮定が満たされない場合、結果解釈に注意が必要となる。
研究用途では、PH 仮定を明示的に検証し、その証拠を示せることが望ましい。

#### 目的
Cox 解析結果の妥当性を確認し、PH 仮定違反の有無をユーザーに明示できるようにする。

#### 実装内容案
- Schoenfeld 残差検定の自動実行
- Diagnostics UI への検定結果表示
- Schoenfeld 残差プロット埋め込み
- PH violation の警告表示
- landmark / RMST など代替解析への導線表示
- レポートへの診断結果記録

#### 研究上の利点
- Cox 結果の信頼性を補強できる
- 査読での PH assumption 指摘に対応しやすい
- 統計モデルの前提確認をユーザーに促せる
