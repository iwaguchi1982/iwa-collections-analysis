# Guardrails（計算の信頼性ガード）

このアプリでは、解析結果の「見た目が出ている」だけでは安全とは限らないため、
**推定が不安定になりやすい条件**をチェックして、色で提示します。

## 表示ルール（必須）

- ✅ **緑：OK**
- ⚠️ **黄：注意（解釈に注意）**
- ⛔ **赤：危険域（計算スキップ/参考値扱い）**

> ⚠️/⛔ の場合は必ずこのページ（Help → Guardrails）を参照してください。

---

## 🚨 Global Guardrails (アプリ全体の安全装置)

データセットを切り替えた直後に、以下の基準を満たさない場合はアプリ全体で強力なアラートが表示されます。

- ⛔ **Event DANGER**: データセット全体の生存イベント（死亡数など）が **10件未満** の場合、生存時間解析としての統計的パワーが全く足りないため、**それ以降のすべての解析タブ（KM, Cox等）がロック（操作不能）**されます。
- ⚠️ **Missing Data CAUTION**: 左サイドバーで選択された共変量（Age, Sex, Stageなど）のいずれかが **20%以上** 欠損している場合、画面上部に警告バナーを表示します。傾向スコアマッチング（PSM）や多変量Cox回帰において大幅なサンプル欠落が起きるため注意してください。

## 典型的な判定項目

### 1) 入力データの必須列

- OS_MONTHS / is_dead
- 1-gene: expression
- 2-gene: expression_1 / expression_2
- 4-group解析: group

**欠落している場合は ⛔（危険域）**
- 解析をスキップ、または「参考値」に落とします。

### 2) 群サイズ（group counts）

**極端に少ない群**があると、
- KMの曲線がギザギザ
- Coxの推定が不安定（CIが極端、p値が変動）

になりやすいです。

目安（運用上の例）：
- ✅：各群 n >= 20
- ⚠️：いずれかの群 10 <= n < 20
- ⛔：いずれかの群 n < 10（またはイベント数が極端に少ない）

> ※閾値はプロダクト要件に応じて調整可能です。

### 3) イベント数（events）

Coxはイベント数が少ないと不安定です。

- ✅：イベント数が十分（例：>= 20）
- ⚠️：イベント数が少なめ（例：10–19）
- ⛔：イベント数が極端に少ない（例：<10）

### 4) Coxの収束・分離（separation）

- ある群でイベントが0
- ある説明変数で完全にイベントが分離

などがあると、Coxが収束しない/極端なHRになることがあります。

- ⚠️：CIが極端に広い、係数が巨大
- ⛔：モデル自体が失敗（例外、収束不能）

### 5) cutoff依存性（Cut smoothness）

cutoffを少し動かしただけで p値が大きく揺れる場合、
**「たまたまそのcutoffだけ有意」** という危険があります。

- ✅：広い範囲で一貫して小さいp
- ⚠️：cutoffによって結果が変わる
- ⛔：ほぼランダムに見える/極端に不安定

## 推奨する読み方（実務）

- ✅：通常通り解釈
- ⚠️：
  - 解釈はするが「不安定さ」を本文に明記
  - cutoff scan / 追加解析で頑健性を確認
- ⛔：
  - そのセクションは**計算スキップ** or **参考値**として扱い、
    “結論”には使わない
