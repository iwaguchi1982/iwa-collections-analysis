### SYS-26-0006）外部コホート取り込みの一般化
#### 背景
Validation 画面では custom CSV / TSV upload が可能だが、実際の研究では GEO、ArrayExpress、自前 RNA-seq など多様な外部データを持ち込むニーズがある。
共同研究や再現性確認を進めるには、取り込み導線をより一般化する必要がある。

#### 目的
多様な外部コホートを簡便に取り込み、独立検証や共同研究データ解析を容易にする。

#### 実装内容案
- GEO / ArrayExpress / 自前 RNA-seq 用テンプレート取り込み
- clinical 列マッピング UI
- 発現データ形式（wide / long）の自動判定支援
- survival 列の必須チェック
- gene ID 変換ログ
- インポート時 QC レポート
- 取り込み後の欠測・件数・分布サマリ表示

#### 研究上の利点
- 独立コホート検証が圧倒的にやりやすくなる
- 共同研究データの持ち込み障壁が下がる
- “外部で再現できるか” を現実的に検証しやすくなる

### SYS-26-0001）多重検定を正面から扱う機能の強化
#### 背景
本システムでは Auto Discovery、1遺伝子 cutoff 探索、2遺伝子 pair 探索など、多数の候補や多数の条件を同時に評価する解析が多い。
このような探索的解析では、名目 p 値のみでは有意性が過大評価されやすく、論文化や査読対応において弱点となる。

#### 目的
探索的解析における偶然の有意結果を過信しないようにし、候補の信頼性をより適切に評価できるようにする。

#### 実装内容案
- BH-FDR による多重検定補正
- permutation ベースの empirical p-value 算出
- 再サンプリングによる stability score 算出
- 1遺伝子 cutoff 探索における cut smoothness 可視化
- 2遺伝子 pair 探索でも同様の補正指標を統一的に表示
- nominal p / adjusted p / empirical p / stability の並列表示

#### 研究上の利点
- 探索的解析の信頼性を高められる
- 査読での「多重比較をどう扱ったか」に答えやすい
- 偶然の当たりと安定した候補を区別しやすくなる

### SYS-26-0004）2遺伝子解析の完成度向上
#### 背景
2遺伝子解析は本システムの大きな特徴の一つであり、単独遺伝子では捉えにくい相互作用や複合バイオマーカー候補を探索できる。
一方で、さらに研究用途へ高めるには、相互作用の解釈や subgroup ごとの再現性などをより厚くする余地がある。

#### 目的
2遺伝子解析を、単なる 4 群比較にとどまらず、相加・相乗効果や再現性評価まで含む研究用機能として完成度を高める。

#### 実装内容案
- 相加効果・相乗効果の定量化
- interaction 項の effect size 要約
- 4群の絶対リスク差表示
- subgroup / subtype 別再現性ヒートマップ
- detailed analysis の不足部分移植
- discovery / validation / meta-analysis との連携強化

#### 研究上の利点
- 複合バイオマーカー解析の説得力が増す
- 2遺伝子の“面白さ”を統計的に裏付けしやすい
- 研究仮説生成から外部検証まで一貫性が出る

### SYS-26-0005）臨床共変量の強化
#### 背景
年齢、性別、病期などの基本的な臨床共変量に加えて、研究用にはより多様な臨床背景因子を柔軟に扱いたい場面が多い。
特にデータセットごとに利用可能な変数が異なるため、固定的な設計では対応しきれない。

#### 目的
データセットごとに利用可能な臨床共変量を自動認識し、適切な候補として提示できるようにする。

#### 実装内容案
- 治療歴、病期詳細、喫煙歴、腫瘍純度、バッチ、採取部位などの検出
- データセットごとに使用可能な共変量の自動候補化
- 列名ゆれの吸収
- 欠測率表示と警告
- 推奨共変量セットの提示
- Cox / PSM / subgroup 解析との統合

#### 研究上の利点
- 現実的な臨床調整がしやすくなる
- データセット差に柔軟に対応できる
- 研究者が使える情報を取りこぼしにくくなる

### YS-26-0009）PH仮定チェックと時間依存効果の正式実装
#### 背景
Cox 比例ハザードモデルは本システムの中心的解析法の一つだが、比例ハザード仮定が満たされない場合、結果解釈に注意が必要となる。
研究用途では、PH 仮定を明示的に検証し、その証拠を示せることが望ましい。

#### 目的
Cox 解析結果の妥当性を確認し、PH 仮定違反の有無をユーザーに明示できるようにする。

#### 実装内容案
- Schoenfeld 残差検定の自動実行
- Diagnostics UI への検定結果表示
- Schoenfeld 残差プロット埋め込み
- PH violation の警告表示
- landmark / RMST など代替解析への導線表示
- レポートへの診断結果記録

#### 研究上の利点
- Cox 結果の信頼性を補強できる
- 査読での PH assumption 指摘に対応しやすい
- 統計モデルの前提確認をユーザーに促せる
