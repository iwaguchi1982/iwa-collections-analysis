# 🔰 生存解析の「よくある落とし穴」と解釈

本アプリは簡単に生存時間解析（Survival Analysis）を行えることを目指していますが、その反面、**「クリックだけで出た結果（特に p値）を鵜呑みにしてしまう危険性」**があります。
このドキュメントでは、陥りやすい解釈の間違い（Pitfalls）と、適切なデータ解釈のためのチェックリストを解説します。

---

## 1. 「p < 0.05 だから意味がある！」という誤解
**最も危険な罠です。** p値はサンプルサイズ（N数）に強く影響されます。

- **落とし穴**: 数千人の PanCancer データ等を使えば、生物学的に無意味な微小な差でも簡単に p < 0.05 になりえます。
- **解釈**: 
  - p値だけでなく、必ず **ハザード比 (Hazard Ratio: HR)** の実数値を確認してください（例：HR が 1.05 など 1 に近すぎる場合は、臨床的な意義は薄いかもしれません）。
  - 各群の **n数 (サンプルサイズ)** と **イベント数 (死亡等の数)** を確認してください。イベント数が一桁の場合、計算された p値やハザード比は極めて不安定です。

## 2. カプランマイヤー曲線が「交差」している場合
**比例ハザード性 (Proportional Hazards Assumption) の欠如**を疑うサインです。

- **落とし穴**: KM曲線（グラフの線）が途中で大きく交差している場合、Cox比例ハザードモデルや Log-rank 検定が前提とする「リスクの比が時間に関わらず一定」という条件が崩れています。
- **解釈**: 前半と後半でリスクが逆転していることを意味します。この場合、出力されたひとつのハザード比だけで全体を語ることはできません。解析手法の変更や、期間を区切った解析（Landmark analysis）などが必要になるケースがあります。

## 3. Auto-Cutoff と p-hacking（過剰最適化）
「一番綺麗に分かれるしきい値」を探すことのリスクを知りましょう。

- **落とし穴**: 何十通りもしきい値（Percentile）をスキャンして最も p値が小さくなる点（Auto-Cutoff）を採用すると、統計学的には**多重検定の問題**が生じ、偶然の産物（偽陽性）を拾っている確率が跳ね上がります（これを p-hacking と呼びます）。
- **解釈**:
  - `Cut smoothness scan` を確認し、「そのしきい値の周辺でも安定して有意差が出ているか（線が滑らかか）」を確認してください。ピンポイントで特定の Percentile だけ p < 0.05 になる場合は、偶然のノイズの可能性が高いです。
  - 論文にする場合、このアプリで見つけた最適カットオフ値は「探索的 (Exploratory)」な結果として扱い、最終的には**独立した別のデータセット（外部コホート）**で同じしきい値を適用して再現性を確認することが求められます。

## 4. 単変量解析と多変量解析の違い（交絡因子）
1-Gene モードなどで遺伝子単独の生存分析を行うと「有意」に出るが、年齢やステージで調整すると「非有意」になるケースがあります。

- **落とし穴**: 「対象遺伝子の発現が高いほど死亡リスクが高い」と思いきや、実は「対象遺伝子の発現が高い人は高齢者が多かっただけで、本当の死亡原因は年齢だった（交絡）」というパターンです。
- **解釈**: 
  - サイドバーにある `Clinical Covariates (Adjustment)` で Age や Stage にチェックバックを入れ、**Adjusted (調整済み) Cox** の結果（Forest Plot等）を確認してください。Adjusted でも有意に残っていれば、その遺伝子が独立した予後因子である可能性が高まります。

## 5. 多重検定の罠 (Auto Discovery) と Traffic Light（信号機）UI
全遺伝子ペアの組み合わせからスキャンを行う Auto Discovery 機能は、非常に強力ですが「数万回のサイコロを振れば、必ずどこかで当たりが出る」という問題があります。

- **解釈のコツ**:
  - **FDR Traffic Light** に従ってください。
    - 🟢 **OK (q < 0.05):** 統計的に有意。論文記載レベルで胸を張れるペアです。
    - 🟡 **Hypothesis (0.05 ≦ q < 0.1):** ギリギリ有意ではないが、傾向として「仮説生成」のネタになります。
    - 🔴 **Danger (q ≧ 0.1):** 偶然の可能性が高いです。これを無理やり採用して「見つけました！」と言うのは厳禁です。
  - 発見されたペアについては、必ず Analysis タブに移行し、個別の癌種 (Stratified 解析) で一貫した傾向が出ているか、少数サンプルによるノイズでないかを確認してください。

## 6. 仮説生成ツールとしての限界（必ず外部コホートで Validation を！）
本アプリ（iwa-collections-analysis）のような探索的ツールから得られた結果は、あくまで**「仮説生成 (Discovery)」**の段階に過ぎません。これを単独で「証明」として論文のメイン結果に据えることは避けてください。

- **落とし穴**: 手元のデータセット（例: TCGA）で最も都合よく当てはまる「最高の p値」や「最適なカットオフ」を探し出す行為は、その特定のデータセットに対する**過学習（Overfitting）**を引き起こしています。
- **解釈（正しい？研究フロー）**:
  1. **Discovery (探索):** 本アプリ等を使い、有望なターゲット遺伝子やスコア、**「固定されたしきい値」** を決定する。
  2. **Locking (ルールの固定):** 「この遺伝子の組み合わせで、スコアが X 以上の人を High とする」というルールを完全に固定し、それ以上パラメータを弄らない。
  3. **Validation (検証):** 完全に独立した外部のデータセット（GEO に登録されている別の臨床データや、自施設の患者データなど）を用意する。
  4. **Testing (最終テスト):** 固定したルールやしきい値をそのまま外部データに当てはめ、そこではじめて計算された生存曲線の p値だけを「真の証明（Confirmatory）」として用いる。

## 7. 高度な解析機能（Advanced Features）を活用した「堅牢性」の確認
本アプリに搭載されている5つの高度な解析機能は、上記のような「落とし穴」を回避し、あなたの仮説（見つけてきた遺伝子）が本物かどうかを検証するために非常に強力な武器になります。

### A. Validation モード（過学習のチェック）
- **活用法**: `1-Gene` や `Auto Discovery` でp値の低い素晴らしい遺伝子を見つけた後、**必ず** `Validation` タブを開いてください。
- **解釈**: Discoveryデータセットで見つけた「しきい値（Percentile）」をValidationデータセットに強制適用したとき、**Validation側でも生存曲線がきれいに分かれ、p < 0.05 になれば大成功**です。もしValidation側でグラフが交差したりp値が全く有意でなかったりした場合、その仮説は「Discoveryデータセットに特有のノイズ（過学習）」だった可能性が高いと解釈します。

### B. Meta-Analysis（全体的な一貫性のチェック）
- **活用法**: 特定の癌種だけでなく、複数のデータセット（コホート）にわたってその遺伝子が普遍的な予後因子であるかを俯瞰します。
- **解釈**: Forest Plotの「全体（総合）の効果」を見るだけでなく、各コホートの **点の位置（左側か右側か）が揃っているか**（＝一貫してリスクを上げているか、下げているか）に注目してください。ある癌では真逆の効果（Cross-over）を示している場合、そのマーカーは「汎がん（Pan-Cancer）」のバイオマーカーとしては不適切である可能性があります。

### C. Propensity Score Matching (PSM)（交絡因子の徹底排除）
- **活用法**: Age（年齢）やStage（進行度）といった強烈な交絡因子が原因で「見かけ上の生存率の差」が出ていないかをチェックします。
- **解釈**: まず **Covariate Balance Check** を見て、マッチング後にすべての変数の SMD が 0.1 未満になっている（背景がきれいに揃った）ことを確認してください。**背景を揃えた後（After Matching）にKM曲線の有意差が消えてしまった場合**、その遺伝子自体には生存への直接的な影響力はなく、実は年齢やステージの影響を見ていただけだった、と結論づけます。

### D. Landmark Analysis（比例ハザード性の崩れへの対策）
- **活用法**: KM曲線が途中で交差している（上述の落とし穴2）場合に適用します。
- **解釈**: 曲線が交差する大体の時期（例：術後24ヶ月）をCutoffとして指定し、「Early Cohort」と「Late Cohort」に分けて解釈します。「術後早期はA群が悪いが、長期的に見るとA群の方が生存率が高い」といった、時間依存的なリスクの逆転（Time-dependent effect）を証明する材料になります。

### E. DEG & PCA（結果からの逆探索）
- **活用法**: 「どの遺伝子を調べればいいか全くわからない」という状態の初期探索（Discovery）に用います。
- **解釈**: Volcano Plotで右上に飛んでいる遺伝子（短期死亡群で有意に高発現）や左上（長期生存群で有意に高発現）の遺伝子は素晴らしい候補ですが、これだけで論文の結論にはできません。ここで見つけた遺伝子候補を記録し、改めて `1-Gene` 解析や `Validation` モードに戻って、独立したデータセットでその効果を緻密に検証するフローを回してください。
